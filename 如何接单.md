# 如何接单

本文基于 `渠道Job搜索清单.md` 和 `渠道Job岗位数统计-2026-05-02.md`，整理新手从软件开发、数据分析、AI Agent / 自动化方向开始接单的实操路径，以及更适合长期发展的供给模式。

## 1. 基本判断

新手不要从“我会写代码”开始接单，而要从“我能交付一个小而明确的业务结果”开始。

相比泛泛竞争 `full stack developer`、`backend developer`、`software developer`，更适合新手切入的关键词是：

- `AI automation`
- `workflow automation`
- `chatbot developer`
- `RAG developer`
- `API integration`
- `internal tools`
- `data analyst`
- `BI`
- `流程自动化`
- `聊天机器人`
- `数据分析`
- `系统开发`

这些关键词更贴近客户愿意付费的业务结果，例如减少人工复制粘贴、自动生成报表、搭建客服机器人、做企业知识库问答、连接多个业务系统等。

## 2. 渠道优先级

`渠道Job搜索清单.md` 中的平台可以分成两类：一类更适合直接接单，另一类更适合观察岗位需求热度。

| 阶段 | 渠道 | 主要目的 |
| --- | --- | --- |
| 第 1 阶段 | Fiverr / 小红书 / 朋友圈 / 国内私域 | 做标准化小服务，拿第一批案例 |
| 第 2 阶段 | Upwork | 找海外明确项目，练习 proposal 和交付边界 |
| 第 3 阶段 | 程序员客栈 / 猪八戒 | 接国内整包或模块项目，但要筛选需求质量 |
| 长期 | LinkedIn / GitHub / 公众号 / 知乎 | 建立信任，获取高客单咨询和企业线索 |
| 不建议新手主攻 | LinkedIn Jobs / Indeed / Wellfound | 更偏招聘岗位，不是最直接的接单入口 |

新手要优先关注 `projects`、`需求`、`Gigs`，而不是只看 `jobs`。LinkedIn、Indeed、Wellfound 适合判断市场热度，但不是最适合当天开始找第一单的渠道。

## 3. 新手如何开始

### 3.1 先选一个细分服务包

不要写“我可以做 AI Agent、全栈开发、数据分析”。更好的方式是把能力写成客户听得懂的结果。

可以从以下服务包开始：

- 把客服 FAQ 文档做成可嵌入网站的 AI 问答机器人。
- 把 Excel / Google Sheets 报表自动生成周报。
- 把表单、邮件、CRM 之间的数据流自动同步。
- 给企业内部文档做一个 RAG 检索问答 Demo。
- 把人工复制粘贴流程改成 Zapier / Make / n8n 自动化。
- 给小团队做一个订单、客户、库存的轻量后台。

新手最好的第一单不是技术最炫，而是边界最清楚、能在 3-7 天内交付、有明确前后对比的项目。

### 3.2 准备 3 个作品

不要等真实客户来了才开始准备案例。新手可以先做 2-3 个模拟真实交付的作品，每个作品都应该包含：

- 一个演示页面或录屏；
- 一份“客户问题 -> 方案 -> 交付结果”的案例说明；
- 一个报价范围；
- 一个交付清单；
- 一个维护选项。

推荐的作品组合：

| 作品 | 对应搜索词 | 可售卖服务 |
| --- | --- | --- |
| FAQ Chatbot | `chatbot developer`、`AI agent` | 网站客服机器人 |
| 文档 RAG Demo | `RAG developer`、`LLM engineer` | 企业知识库问答 |
| 自动报表工具 | `workflow automation`、`data analyst` | 数据清洗、报表自动化 |

### 3.3 从低风险报价开始

新手不建议一开始按小时卖，因为客户很难判断开发者是否值得信任。更适合的方式是固定价套餐。

| 套餐 | 价格定位 | 内容 |
| --- | --- | --- |
| 诊断包 | 低价或免费 | 30-60 分钟需求梳理，输出流程图和报价 |
| Starter | 低价 | 单一功能 Demo / 小自动化 |
| Standard | 中价 | 可上线版本，含部署和文档 |
| Advanced | 高价 | 多系统集成、权限、日志、培训、维护 |

Fiverr 更适合这种套餐化供给；Upwork 则适合在 proposal 里把交付拆成里程碑。

### 3.4 每天固定搜索少量关键词

不要海投。每天看 5-10 个高相关项目即可，优先筛选：

- 提案数少于 5 或 5-10；
- 需求描述具体；
- 客户有付款记录；
- 项目目标明确；
- 预算不是极低；
- 需求包含 `automation`、`integration`、`chatbot`、`RAG`、`internal tool` 等关键词。

对新手来说，`AI agent` 这个词流量大但可能很泛；`AI automation`、`workflow automation`、`API integration` 反而更容易成交，因为客户通常已经知道自己要解决什么流程问题。

## 4. 好的供给模式

好的供给模式不是“有人发需求，我去抢单”，而是形成一个可复制的交付货架。

```text
引流服务：诊断 / 需求梳理
        ↓
标准小单：Bot / RAG Demo / 自动化脚本 / 报表
        ↓
定制项目：系统集成 / 内部工具 / 多流程自动化
        ↓
长期收入：维护 / 数据更新 / Prompt 优化 / 日志巡检 / 月度托管
```

### 4.1 服务产品化

不要只卖“开发时间”，而要把能力包装成明确商品。

较弱的表达：

> 我会 Python、React、LangChain、OpenAI API。

更好的表达：

> 我帮小型电商把订单邮件、表格和客服回复自动化，3 天内交付一个可运行流程。

### 4.2 交付模块化

把常见项目拆成固定模块：

- 需求诊断；
- 数据 / 文档整理；
- API 接入；
- Bot / Agent / RAG 实现；
- 后台管理；
- 部署上线；
- 使用培训；
- 月度维护。

模块化之后，报价更清楚，也方便复用。

### 4.3 案例可视化

客户通常不想先看代码，而是想看结果。每个服务最好都配备：

- 截图；
- 录屏；
- 前后对比；
- 节省时间估算；
- 交付周期；
- 可选升级项。

### 4.4 低门槛进入，高客单升级

第一单可以小，但不能永远停留在小单。理想路径是：

```text
$99-$300 小诊断 / Demo
→ $500-$1500 小型自动化 / Bot
→ $2000-$8000 内部工具 / RAG / 多系统集成
→ 月费维护 $200-$1000+
```

中文市场也类似：

```text
几百元需求梳理
→ 3000-10000 元小工具 / 自动化 / 数据报表
→ 2万-10万元系统整包 / AI 改造
→ 月度托管维护
```

### 4.5 从单点工具走向业务流程

长期更有价值的不是“做一个聊天机器人”，而是把 AI 和自动化嵌入具体业务流程，例如：

- 客服流程自动化；
- 销售线索处理自动化；
- 企业知识库问答；
- 财务 / 运营报表自动化；
- 订单 / 库存 / CRM 流程集成；
- 内部审批和文档处理 Agent。

也就是从 `AI Agent` 这个技术词，转成客户愿意付费的业务词。

## 5. 30 天起步方案

| 周期 | 任务 |
| --- | --- |
| 第 1 周 | 选 1 个方向：AI 客服、RAG 知识库、流程自动化、数据报表四选一 |
| 第 2 周 | 做 2-3 个 Demo，写成案例页，准备中英文服务描述 |
| 第 3 周 | 上 Fiverr / Upwork / 国内平台，每天投 5 个高相关项目 |
| 第 4 周 | 把客户问题整理成 FAQ，优化服务包、报价和 proposal 模板 |

最推荐的新手切口是：

> AI 自动化 + RAG / Chatbot + API integration 的小型交付包。

这个方向同时满足三个条件：需求在渠道清单里高频出现，交付边界容易缩小，后续容易升级成系统集成和月度维护。相比泛软件开发，它更容易做出差异化；相比大型 AI Agent，它不会一开始就把复杂度拉得过高。
