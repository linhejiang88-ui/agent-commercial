# GitHub 机会搜索清单

## 统一规则

- 目标不是找 star 最高的项目，而是找“增长快、issue 多、商业化弱、可服务化”的项目。
- 每条机会必须至少保留 repo URL、stars、forks、open issues、最近更新时间、topics、license 和评分。
- 优先看 AI 自动化、RAG / 文档自动化、Agent 框架、Chrome / Shopify / Slack 插件、n8n / Make workflow。
- 不把开源项目代码直接复制成产品；优先做部署、集成、托管、模板、中文教程、行业方案和运维服务。

## 评分模板

| 维度 | 评分问题 | 1 分 | 3 分 | 5 分 |
| --- | --- | --- | --- | --- |
| 痛点强度 | issue 是否真实、反复、具体 | 泛泛讨论 | 有具体问题 | 多个用户反复卡在同一问题 |
| 付费可能 | 是否有人问 hosted / pricing / enterprise | 无 | 有间接信号 | 明确问托管、企业版、报价 |
| 竞争密度 | 是否已经红海 | 大量成熟 SaaS | 有差异化空间 | 垂直需求明确且竞品少 |
| 交付可行性 | 3 人团队是否能做 | 超出能力 | 30-60 天可 PoC | 30 天内可做首单 |
| 资产复用 | 是否能沉淀模板 / SOP / workflow | 一次性项目 | 可复用组件 | 可形成模板包或托管产品 |

18 分以上进入候选机会池。
