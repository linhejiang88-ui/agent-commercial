## 2. 文档生成参考的核心分析点

### 2.1 市场规模与增长

- 软件开发外包仍是最大基本盘，但增长较稳，通用开发和人天外包会持续承压。
- AI Agent 是增速最快的技术交付方向，适合小团队做单场景、单流程、单部门的项目。
- AI 咨询服务不应只理解为战略报告，而应采用 `咨询 + 实施 + 托管运营` 口径。
- 中国服务外包市场中，ITO 仍有体量但短期承压，BPO 韧性更强，KPO 适合与知识增强和数据分析结合。

### 2.2 机会点排序

1. **最大机会：AI 咨询 + AI Agent 实施 + 托管运营。**
   - 推荐服务包：`AI 诊断 + 场景筛选 + 单场景 PoC + Agent 实施 + 月度托管优化`。
   - 逻辑：咨询建立信任，PoC 降低决策风险，实施和维护形成持续收入。

2. **次优机会：BPO 流程自动化 + 存量系统 AI 化改造。**
   - 典型场景：客服、销售线索、工单流转、审批预处理、运营助手。
   - 逻辑：客户已有系统、数据和流程，ROI 更容易解释。

3. **可做机会：AI 辅助数据分析 / KPO 增强。**
   - 典型场景：知识整理、经营看板、预测建模、决策辅助、投标和合规检索。
   - 逻辑：适合作为 Agent 项目的上游数据治理和下游效果评估模块。

4. **低优先级：纯软件开发、人天外包、简单 Bot。**
   - 逻辑：竞争强、复购弱，且 AI Coding 会继续压缩标准化开发溢价。

### 2.3 渠道与获客判断

- Upwork、Fiverr 等国际平台适合验证海外 AI 自动化、RAG、内部工具和数据流程需求。
- 国内众包平台适合试单和价格锚定，但不适合作为长期高利润主渠道。
- 私域转介绍、本地线下活动和老客户复购更适合高信任、高客单价项目。
- 内容营销、GitHub、LinkedIn、小红书等适合长期建立专业信任，不应只看短期成交。

### 2.4 产品化与收费方式

- 最优交付链路：`AI 诊断 -> 场景筛选 -> ROI 评估 -> PoC / MVP -> Agent / RAG 实施 -> 上线培训与治理 -> 月度托管优化`。
- 推荐收费结构：`诊断费 + PoC 实施费 + 上线部署费 + 月度维护费`。
- AI Agent 项目不能只看开发成本，还要纳入部署、权限、日志、向量库、模型调用、token 成本和持续优化。

### 2.5 AI Coding 对外包业务的影响

- ITO：标准化开发、基础编码和低门槛维护被压价，系统集成和行业定制仍有价值。
- BPO：流程执行从人工外包转向人机协同运营。
- KPO：低端资料整理和简单摘要被替代，高验证密度知识服务门槛提高。
- 小团队壁垒不在“写代码更快”，而在业务理解、系统集成、评估验收和持续运营能力。
