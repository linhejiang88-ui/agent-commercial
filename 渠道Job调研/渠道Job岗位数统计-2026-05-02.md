# 渠道 Job 岗位 / 项目数量统计（2026-05-02）

统计依据：基于 `渠道Job搜索清单.md` 的平台和关键词，在 2026-05-02 通过公开网页、搜索结果索引页和平台公开分类页收集。  
重要限制：Upwork、Indeed、LinkedIn、Wellfound 的实时站内搜索在本地 `curl` 环境中存在 Cloudflare / WAF / 地区跳转 / 登录态限制；因此本表只记录可公开复核的数字。不能公开验证的关键词不填推测值。

## 统计口径

- `官方公开页总数`：平台页面直接显示的结果数，优先采用。
- `公开索引代理数`：搜索引擎索引到的平台页标题或摘要中显示的数量，可作热度参考，但需要进入平台登录态复核。
- `不可公开获取`：当前环境无法从公开页面读取总数，需人工登录 App 或网页按原清单规则复核。
- LinkedIn / Indeed / Wellfound 统计 `jobs`；Upwork / 程序员客栈 / 猪八戒统计 `projects` 或 `需求数`；Fiverr 只作为 `services proxy`。

## 复核建议

1. 对 Upwork、LinkedIn、Indeed：使用登录态浏览器或 App，逐个关键词记录公开总数，并单独筛选 `Remote`、`Contract`、`Intermediate / Expert`。
2. 对程序员客栈、猪八戒：必须先进入真实“项目 / 需求”视图，避免把开发者主页、案例页、服务商商品页计入需求数。
3. 对 Fiverr：只作为服务供给热度，不与 jobs / projects 直接横向比较；`AI agent` 与 `AI agents` 聚合页高度重复，不能相加。
4. 所有复核结果建议补充 `前2页可见量` 和 `高相关前屏数量`，用于判断“数量大但低相关”的关键词。
