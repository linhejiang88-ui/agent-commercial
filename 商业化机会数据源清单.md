# 商业化机会数据源清单

本文用于补充现有 `渠道 Job` 与 `X 内容调研` 之外的数据源，目标是找到更适合 3 人技术团队商业化的 AI / AI 工具机会。判断重点不是“哪个方向热”，而是同时验证：用户痛点、付费意愿、竞争密度、交付可行性和资产复用空间。

## 1. 数据源优先级

| 优先级 | 数据源 | 主要验证 | 适合发现的商业化机会 | 采集方式 |
| --- | --- | --- | --- | --- |
| P0 | Reddit | 用户痛点、求推荐、替代方案、预算抱怨 | 小企业 AI 自动化、垂直工具、内容产品、人工流程替代 | 官方 API / 人工抽样；不做高频爬取 |
| P0 | Hacker News | 开发者和创业者对工具的真实评价 | Developer tools、AI agent infra、开源转商业 | HN Algolia 搜索 API / 人工复核 |
| P0 | GitHub | 开源热度、开发者采用速度、可复用组件 | n8n workflow、Agent 框架、RAG 工具、模板库 | GitHub REST Search / Web 搜索 |
| P0 | Product Hunt | 新产品定位、投票、评论、竞品话术 | AI 微 SaaS、插件、模板、Newsletter、SaaS idea | Product Hunt API / 人工抽样 |
| P1 | Acquire.com / Flippa | 已有小产品收入、售价、行业、技术栈 | 小 SaaS、内容站、插件、模板站 | 账号内人工抽样；保留截图和字段 |
| P1 | G2 / Capterra | 企业软件差评、替代需求、集成痛点 | B2B AI 自动化、垂直 SaaS、集成服务 | API / 公开评论抽样 |
| P1 | Google Trends / Keyword Planner | 搜索需求、地域、季节性、相关关键词 | SEO 独立站、内容产品、获客主题 | 人工导出 / Ads API |
| P2 | Stack Overflow / Stack Exchange | 开发者卡点和技术需求频率 | API 集成、自动化脚本、开发工具、教程产品 | Stack Exchange API / Web 搜索 |
| P2 | Chrome / Shopify / Slack / Atlassian Marketplace | 插件生态、评分、评论、竞品密度 | 浏览器插件、Shopify AI 工具、团队协作 bot | 公开页面抽样 / 平台 API |

## 2. 五类信号

### 2.1 需求信号

需求信号回答“谁在痛、痛在哪里”。优先看 Reddit、HN、Stack Overflow 和 Google Trends。

| 数据源 | 关键字段 | 高价值样本特征 |
| --- | --- | --- |
| Reddit | subreddit、标题、正文、评论数、upvote、发布时间、链接 | 出现 `I need`、`looking for`、`alternative`、`how do I automate`、`too expensive` 等表达。 |
| Hacker News | 标题、URL、points、comments、created_at、comment text | 评论区讨论工具缺陷、迁移成本、定价争议、开源替代。 |
| Stack Overflow | tag、question、answer count、score、accepted answer | 高频集成问题、API 限制、部署失败、自动化脚本需求。 |
| Google Trends | 关键词、地区、相关查询、时间趋势 | 相关查询持续上升，且能映射到可交付服务或内容资产。 |

### 2.2 付费信号

付费信号回答“有没有人已经为类似东西付钱”。优先看 Acquire.com、Flippa、G2 / Capterra 和 AppSumo。

| 数据源 | 关键字段 | 高价值样本特征 |
| --- | --- | --- |
| Acquire.com / Flippa | revenue、profit、asking price、multiple、category、tech stack | 有真实收入、利润、用户数或出售价格，而不是只有流量故事。 |
| G2 / Capterra | rating、review title、pros、cons、company size、category | 差评集中在价格、集成、自动化、客服、报表、权限等可服务化问题。 |
| AppSumo | deal price、reviews、Q&A、use case、rating | 低价产品仍能收获大量评论，说明市场愿意为简单工具付费。 |

### 2.3 产品信号

产品信号回答“别人如何包装和定位”。优先看 Product Hunt、Chrome Web Store、Shopify App Store。

| 数据源 | 关键字段 | 高价值样本特征 |
| --- | --- | --- |
| Product Hunt | tagline、topics、votes、comments、maker、launch date | 定位清晰、评论里出现真实用例、产品能映射到小团队可交付能力。 |
| Chrome Web Store | category、rating、reviews、users、last updated | 用户数高但差评集中，适合做替代插件或服务包。 |
| Shopify App Store | pricing、reviews、merchant pain、category | 商家愿意持续付费，且需求垂直明确。 |

### 2.4 供给信号

供给信号回答“有没有可复用资产”。优先看 GitHub、模板库、开源 workflow。

| 数据源 | 关键字段 | 高价值样本特征 |
| --- | --- | --- |
| GitHub | stars、forks、created_at、pushed_at、topics、issues、license | 新项目增长快、issues 多、商业支持少，适合做实施、托管或模板化。 |
| n8n / Make 社区模板 | workflow 名称、使用场景、集成应用、复杂度 | 能直接改造成行业服务包，如 lead gen、CRM sync、客服工单。 |
| Hugging Face / Replicate | downloads、likes、model card、license | 模型能力已成熟，但业务落地缺封装和评估。 |

### 2.5 渠道信号

渠道信号回答“哪里能触达客户”。现有仓库已覆盖 Upwork、Fiverr、LinkedIn、X；后续可补 Reddit、Newsletter 和垂直社区。

| 数据源 | 当前状态 | 适合动作 |
| --- | --- | --- |
| Upwork / Fiverr | 已有公开数和截图证据 | 用于验证关键词需求池和服务包装方式。 |
| X | 已有 AI 赚钱分享分析 | 用于观察个人服务商如何包装机会和导流。 |
| LinkedIn | 已有岗位数信号 | 用于反向设计内容营销和企业服务包。 |
| GitHub | 已完成第一轮 Search API 粗筛，见 [`GitHub商业化机会分析-2026-05-03.md`](GitHub机会调研/GitHub商业化机会分析-2026-05-03.md) | 用于发现可复用开源资产、部署服务、托管和模板化机会。 |
| Reddit / HN | 待补 | 用于找到真实抱怨和未被平台服务页覆盖的机会。 |

## 3. 机会打分模板

每条机会用 5 个维度打分，满分 25 分。建议只把 `18 分以上` 的方向写入主报告或执行路线。

| 维度 | 评分问题 | 1 分 | 3 分 | 5 分 |
| --- | --- | --- | --- | --- |
| 痛点强度 | 用户是否明确表达痛点 | 泛泛兴趣 | 有具体问题 | 有强烈抱怨、失败成本或反复求助 |
| 付费证据 | 是否有人付费 | 无收入线索 | 有价格 / 预算 | 有收入、交易、订阅或明确购买 |
| 竞争密度 | 是否已经红海 | 强同质化 | 有差异化空间 | 竞品少且需求明确 |
| 交付可行性 | 3 人团队是否能做 | 超出能力 | 30-60 天可做 PoC | 30 天内可交付首单 |
| 资产复用 | 是否能沉淀模板 / 产品 | 一次性项目 | 可复用部分组件 | 可形成 SOP、模板、工作流或微 SaaS |

## 4. 推荐采集顺序

1. **Reddit + HN：** 先找真实痛点和抱怨，不急着看产品。
2. **GitHub：** 找可复用开源项目、workflow、模板和增长中的工具。
3. **Product Hunt：** 看竞品如何定位、评论区是否出现真实使用场景。
4. **Acquire.com / Flippa：** 验证小产品是否真的有收入。
5. **G2 / Capterra：** 从差评中反推 B2B 服务包。
6. **Google Trends / Keyword Planner：** 判断是否值得做 SEO / 内容获客。

## 5. 第一批建议主题

| 主题 | 为什么值得查 | 推荐数据源 |
| --- | --- | --- |
| n8n / Make AI 自动化服务 | X 和 Upwork 都显示自动化信号强，适合现金流 | Reddit、HN、GitHub、Upwork |
| AI receptionist / appointment automation | 场景垂直、ROI 清晰，适合本地服务行业 | Reddit、Product Hunt、G2 |
| AI SEO / programmatic SEO | 有高收入案例也有失败反例，适合验证 | HN、Reddit、Google Trends、Acquire.com |
| AI Newsletter / 内容产品 | 启动快，适合作为获客前端 | Product Hunt、X、Acquire.com |
| Chrome extension with AI | 小团队可做，分发明确 | Chrome Web Store、Product Hunt、GitHub |
| Shopify AI 工具 | 商家付费意愿强，场景垂直 | Shopify App Store、G2、Reddit |

## 6. 采集约束

- 不绕过登录墙、付费墙、反爬或平台访问限制。
- 对 Reddit、X、G2 等社区内容，只做低频公开数据采集和人工抽样。
- 对收入、MRR、ARR、售价等强结论，必须保留原始链接、截图和采集日期。
- 报告正文只写归纳结论，原始帖子、评论、截图和 JSON 放入对应调研目录。
